# 1. Why is Java called a platform-independent language?

Java is called platform-independent because Java code can run on any operating system that has a JVM (Java Virtual Machine).

**Interview Answer:**
> Java follows "Write Once, Run Anywhere (WORA)" because bytecode can run on any JVM.

---

# 2. Explain the complete Java execution flow.

```text
Java Source Code (.java)
           ↓
      Compiler (javac)
           ↓
       Bytecode (.class)
           ↓
           JVM
           ↓
    Machine Code
           ↓
        Output
```

**Interview Answer:**
> Java source code is compiled into bytecode, and the JVM converts bytecode into machine code for execution.

---

# 3. Difference between JDK, JRE, and JVM

| Term | Meaning | Purpose |
|--------|---------|---------|
| JDK | Java Development Kit | Used to develop Java programs |
| JRE | Java Runtime Environment | Used to run Java programs |
| JVM | Java Virtual Machine | Executes bytecode |

### Easy Formula

```text
JDK = JRE + Development Tools
JRE = JVM + Libraries
```

---

# 4. What is bytecode?

Bytecode is the intermediate code generated after compiling a Java program.

Example:

```text
Hello.java
     ↓
javac
     ↓
Hello.class (Bytecode)
```

**Interview Answer:**
> Bytecode is platform-independent code generated by the Java compiler and stored in a .class file.

---

# 5. Why does Java use bytecode?

Java uses bytecode to achieve platform independence.

The same bytecode can run on any operating system with a JVM.

**Interview Answer:**
> Bytecode allows Java programs to run on multiple platforms without recompilation.

---

# 6. Why is JVM platform dependent?

JVM interacts directly with the operating system and hardware.

Therefore:

- Windows has a Windows JVM.
- Linux has a Linux JVM.
- Mac has a Mac JVM.

**Interview Answer:**
> JVM is platform dependent because it must understand the underlying operating system and hardware.

---

# 7. Why is bytecode platform independent?

Bytecode is not tied to any specific operating system.

Any JVM can understand and execute the same bytecode.

**Interview Answer:**
> Bytecode is platform independent because it can run on any JVM regardless of the operating system.

---

# 8. What happens internally when we compile a Java program?

When we compile:

```bash
javac Hello.java
```

Steps:

1. Compiler checks syntax errors.
2. Compiler checks semantic errors.
3. Source code is converted into bytecode.
4. `.class` file is generated.

**Interview Answer:**
> During compilation, the Java compiler checks the code and converts it into bytecode stored in a .class file.

---

# 9. What happens internally when we run a Java program?

When we run:

```bash
java Hello
```

Steps:

1. JVM loads the class.
2. Bytecode Verifier checks security.
3. Interpreter reads bytecode.
4. JIT Compiler converts frequently used code into machine code.
5. CPU executes the machine code.

**Interview Answer:**
> JVM loads, verifies, and executes bytecode by converting it into machine code.

---

# 10. Why is Java called both compiled and interpreted language?

Java uses both compilation and interpretation.

### Compilation

```text
.java → .class
```

Performed by:

```text
javac Compiler
```

### Interpretation

```text
.class → Machine Code
```

Performed by:

```text
JVM
```

**Interview Answer:**
> Java is called both compiled and interpreted because source code is first compiled into bytecode and then interpreted/executed by the JVM.
